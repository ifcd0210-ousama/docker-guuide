Phase 1: Docker Foundations (Weeks 1–4)

Installation on all OSes, images vs containers, Dockerfile best practices, and everything you need to build your first containerized applications.

Week 1

Installation & Core Concepts

Docker Desktop 4.28+ is recommended for beginners. Here's how to install on Linux:

bash# Linux (Ubuntu/Debian)
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh
sudo usermod -aG docker $USER
# Log out and back in

# Verify installation
docker --version
# Should show: Docker version 25.0.3, build 4debf41

# Start Docker Desktop with AI Assistant
docker desktop start --enable-ai-assist
Docker Desktop's AI Assistant (launched Aug 2025) can explain every command you run. Enable it in Settings → AI Features → "Explain Docker Commands."
Core Concepts

Don't Memorize — Understand

Dockerfile = Build instructions
Registry = Image storage

Week 2

Your First Dockerfile (2026 Best Practices)

# Use specific version tags (never 'latest' in prod)
FROM node:20.11-alpine3.19 AS base

WORKDIR /app

# Copy package files first (better layer caching)
COPY package*.json ./

RUN npm ci --only=production && \
    npm cache clean --force

COPY . .

# Create non-root user (security best practice)
RUN addgroup -g 1001 -S nodejs && \
    adduser -S nodejs -u 1001

USER nodejs

EXPOSE 3000

HEALTHCHECK --interval=30s --timeout=3s \
  CMD node healthcheck.js || exit 1

CMD ["node", "server.js"]
Week 3

Essential Docker Commands

# Build with BuildKit (default in 2026)
docker build -t myapp:1.0 .

# Build with AI-suggested optimizations
docker build --ai-optimize -t myapp:1.0 .

# Run a container
docker run -d -p 8080:3000 --name myapp myapp:1.0

# View logs
docker logs -f myapp

# Execute commands inside container
docker exec -it myapp /bin/sh

# Clean up everything
docker system prune -a --volumes
Pro Tip 2026
Multi-stage builds are now standard practice. They reduce image size by 60-80%. Docker Scout will warn you if you're not using them.

Week 6

Security with Docker Scout

# Enable Docker Scout
docker scout enroll

# Scan for vulnerabilities
docker scout cves myapp:1.0

# AI-powered fix recommendations
docker scout recommendations myapp:1.0

# Auto-fix critical vulnerabilities (NEW)
docker scout quickview myapp:1.0 --auto-fix
Pro Tip 2026
Docker Scout now uses AI to prioritize CVEs based on your actual code usage, reducing false positives by 75%.
Week 8

CI/CD with GitHub Actions

name: Docker Build & Deploy

on:
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v4
    
    - name: Set up Docker Buildx
      uses: docker/setup-buildx-action@v3
    
    - name: Build and scan
      uses: docker/build-push-action@v5
      with:
        context: .
        push: true
        tags: ghcr.io/${{ github.repository }}:${{ github.sha }}
    
    - name: Docker Scout CVE Scan
      uses: docker/scout-action@v1
      with:
        command: cves
        only-severities: critical,high

Week 9

Image Optimization (2026 Techniques)

# Multi-stage build with layer optimization
FROM node:20-alpine AS builder
WORKDIR /app
COPY package*.json ./
RUN npm ci
COPY . .
RUN npm run build && npm prune --production

FROM node:20-alpine AS production
RUN apk add --no-cache dumb-init
WORKDIR /app
COPY --from=builder /app/dist ./dist
COPY --from=builder /app/node_modules ./node_modules
USER node
EXPOSE 3000
CMD ["dumb-init", "node", "dist/server.js"]
Docker Build Cloud (launched 2025) can reduce build times from 10 minutes to 12 seconds. Free for up to 50 build minutes/month in 2026.

  --builder cloud-yourorg \
  --platform linux/amd64,linux/arm64 \
  -t myapp:latest --push .
